# CO2RR-GNN-LLM
This repository contains the workflow used in this work for **CO₂ reduction reaction (CO₂RR) catalyst discovery**, combining **Graph Neural Networks (GNN)** and **Large Language Models (LLM)**.

The repository is divided into two main components:

- **GNN-based catalyst screening workflow**
- **LLM-based catalyst design module**

This section describes the **GNN workflow** used for high-throughput catalyst screening.

---

# GNN-based Catalyst Screening Workflow

The GNN pipeline automatically generates adsorption structures, performs structure optimization, extracts adsorption energies, and evaluates structural descriptors.

The workflow consists of **five main steps**.

---

# Step 1. Slab Preparation

Surface slab structures are obtained from the Materials Project database.

Two approaches can be used:

1. Download bulk structures from Materials Project and construct surface slabs by expanding the supercell and cleaving the surface.
2. Prepare slab structures locally.

The slab preparation scripts are **not included in this repository**.

The final directory should contain slab structures in the following format:
Slab_Material/
POSCAR


---

# Step 2. Generate Adsorption Structures

Adsorption structures are generated by adding CO₂RR intermediates and hydrogen species onto the slab surface.

Scripts used:
slab_add_CO2RR.py
slab_add_H.py


These scripts automatically:

- enumerate adsorption sites
- place adsorbates on the surface
- generate initial adsorption configurations

Example structure:
Slab_CO/
Metal_surface/
CO_01/
POSCAR
CO_02/
POSCAR


---

# Step 3. Structure Optimization Using the GNN Model

Structure optimization is performed using the trained **GNN potential model**.

Scripts:
GNN_model_Cuda_opt.py
GNN_model_Cuda_opt_slab.py


Features:

- supports **GPU and CPU**
- limits the number of optimization steps
- used to efficiently approximate adsorption energies

Optimization is performed for:

- clean slab structures
- slab + adsorbate structures

---

# Step 4. Energy Extraction

After optimization, adsorption energies are extracted automatically from optimized POSCAR files.

Scripts:
energys_poscar_H.py
energys_poscar_ORIGIN.py
energys_poscar_slab.py
energys_poscar_slab_ads.py


These scripts:

- scan the directory structure
- extract energies from filenames
- generate structured energy tables

Example output:
energy_table.csv

---

# Step 5. Chemical Descriptor Analysis

To evaluate intermediate stability and bonding characteristics, structural descriptors such as bond lengths and metal interactions are calculated.

Scripts:
COCO_ana_multiCPU.py
CO_ana_multiCPU.py


These scripts analyze:

- C–C bond distances
- C–metal interaction distances
- adsorption structure stability

The scripts support **multi-CPU parallel analysis** and generate filtered structure tables.

---

# Workflow Summary

The full GNN pipeline is summarized as:
Slab Preparation
↓
Adsorbate Generation
↓
GNN Structure Optimization
↓
Energy Extraction
↓
Chemical Descriptor Screening


This workflow enables **automated large-scale catalyst screening for CO₂RR**.

# LLM-based Literature Mining and Rule Discovery Workflow

This section describes the **LLM-based workflow** used for extracting synthesis knowledge from literature, organizing the extracted rules, and discovering rule families through semantic clustering and visualization.

The pipeline converts scientific literature into **structured rule datasets**, clusters similar rules, identifies representative rules, and visualizes the rule landscape.

The workflow consists of **six main steps**.

---

# Step 1. Literature Collection and PDF to TXT Conversion

Domain-specific literature is first collected manually.

PDF files are converted into **plain text (TXT)** format for downstream processing.  
The PDF-to-TXT conversion step is not included in this repository.

The pipeline assumes the following input format:
literature_txt/
paper_001.txt
paper_002.txt
paper_003.txt


---

# Step 2. Extract Structured Information Using LLM

Scientific information is extracted from the literature using an LLM with a customizable prompt.

Script used:
gpt5_alloy_method_extractor.py


This script:

- reads literature TXT files
- applies a user-defined prompt
- extracts structured information such as synthesis rules or reaction principles

The prompt can be modified for different tasks, making the script reusable for other literature mining problems.

Example output format:
extraction_output/
paper_001_rules.txt
paper_002_rules.txt


The extracted content typically contains **JSON-like rule blocks generated by the LLM**.

---

# Step 3. Convert Extraction Results to JSONL

The extraction results are converted into a structured JSONL dataset.

Script used:
extract_json_rules_to_jsonl.py


This script:

- parses JSON-like rule blocks from TXT files
- performs strict JSON extraction
- applies fallback recovery for partially valid JSON
- assigns global rule IDs and metadata

Output:
syn_3.jsonl


Each line corresponds to one structured rule entry.

---

# Step 4. Semantic Clustering of Extracted Rules

The extracted rules are embedded and grouped into semantic clusters.

Script used:
cluster_synthesis_rules.py


This script:

- converts rules into sentence embeddings
- reduces embedding dimensionality using UMAP
- clusters rules using HDBSCAN

Output:
rule_clusters.csv


This file assigns a cluster ID to each rule.

Cluster IDs represent **automatically discovered rule families**.

---

# Step 5. Representative Rule Identification and Visualization

After clustering, representative rules and visualization data are generated.

Scripts used:
find_cluster_representative_rules.py
generate_rule_umap_visualization.py
plot_rule_umap.py


Functions:

**Representative rule selection**

find_cluster_representative_rules.py

- identifies cluster medoids
- selects the most representative rules in each cluster

Output:
cluster_representative_rules.csv


**UMAP coordinate generation**

generate_rule_umap_visualization.py

- projects rule embeddings into 2D space
- prepares visualization coordinates

Output:
rule_umap_vis.csv

**Visualization plotting**

plot_rule_umap.py

- generates a publication-style UMAP scatter plot
- displays rule clusters and noise points

Output:
Figure_rule_umap.png


---

# Step 6. Rule Dataset Generation and Model Training

After rule extraction and clustering, the rule corpus can be expanded into a **training dataset for LLM fine-tuning**.

Using prompt-based generation, additional training samples are produced from the extracted rules.

These datasets can then be used for:

- **Supervised Fine-Tuning (SFT)**
- **Direct Preference Optimization (DPO)**

Training can be performed using the **OpenAI fine-tuning API** or other LLM training frameworks.

This step enables the construction of **domain-specific reasoning models** trained on the extracted synthesis rules.

---

# Additional Script: Literature Filtering

Script:
ana_json_select_article.py


This script helps filter and select relevant articles from the literature corpus.

Typical uses include:

- removing irrelevant papers
- selecting domain-specific studies
- constructing a clean literature dataset before extraction
